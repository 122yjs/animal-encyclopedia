## 6. 기술적 부채 및 유의사항

### 6.1 `no-question.html` 동기화
- 현재 `index.html`과 `no-question.html`이 **수동 동기화** 상태
- PR #8에서 `no-question.html`도 함께 수정되었으나(63라인 변경), 향후 `index.html` 단독 수정 시 어긋날 위험 존재
- **조치**: 3.7.3 자동 생성 구현 또는 최소한 변경 시 양쪽 동시 수정 체크리스트 운영

### 6.2 이미지 내장 준비 상태
- `scripts/build.js`의 `hasLocalImageAssets()`는 `images/thumbs/` 또는 `images/details/` 폴더 내 이미지 파일 존재 여부를 검사
- `app.js`의 `getImageSource()`는 `config.localImages.enabled`가 true일 경우 내장 경로 우선 사용하도록 준비되어 있음(추정)
- **조치**: 이미지 에셋 확보 후 `images/` 폴더 생성만으로 빌드 파이프라인이 자동 처리

### 6.3 효과음 브라우저 호환성
- Web Audio API는 iOS Safari에서 **사용자 제스처 이후**에만 초기화 가능
- 현재 `ensureAudioContext()`는 `playSound()` 호출 시마다 확인하므로, 첫 터치 이후 정상 작동
- **유의**: 온보딩 "시작하기" 버튼이 첫 제스처 역할을 하므로 문제 없음

### 6.4 용량 현황
- 코드(`app.js` + `styles.css` + HTML): ~150KB
- 효과음: 0KB (Oscillator 합성음, 파일 없음)
- 이미지(예상): +1.5MB (내장 시)
- **총 배포 예상**: ~1.7MB (GitHub Pages 로드 타임 3초 이내 유지 가능)

## 7. 종합 평가

**PR #8 (`codex/featurerefactoring`)은 2차 개선계획의 핵심 항목 대부분을 훌륭하게 구현했습니다.**

- **학습 동기**: 지역별 마일스톤 + 보상 모달로 54마리라는 거대한 목표를 4단계로 세분화하여 지속적인 성취감 제공
- **사용성**: 온보딩 튜토리얼 + 상단 고정 진행도(`stickyProgress`) + 안전 약속 접기(`<details>`)로 시작 화면 복잡도 해소
- **몰입도**: Web Audio API Oscillator 합성음으로 외부 파일 의존 없이 효과음 구현 (용량 0 증가, 정적 사이트 제약 극복)
- **안정성**: 검색 디바운스 + `safeSetStorage()`로 localStorage 안정성 강화

**가장 시급한 남은 작업은 이미지 내장(3.6)입니다.** `build.js`는 이미 `images/` 폴더를 `dist/`로 복사하도록 준비되어 있으므로, 54마리 동물 사진을 `images/thumbs/`와 `images/details/`에 배치하기만 하면 빌드 파이프라인이 자동으로 처리합니다. 이 작업 하나로 방화벽/느린 네트워크 환경에서의 사용자 이탈 문제가 해결될 것입니다.

외부 링크 UX(3.4)와 게임 토큰 확대(3.7.4)는 각각 30분 내외로 해결 가능한 저항의 작업(Low-hanging fruit)이므로, 이미지 내장 작업과 병렬로 진행하면 좋겠습니다.
