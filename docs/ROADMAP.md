## 1. 이번 개정의 핵심 판단

기존 개선 계획은 모바일 UX, 접근성, 문서 정리, 배포 안정성 중심으로 정리되어 있었다.
그러나 실제 사용 중 지역 미션 흐름에서 학생이 다음 미션으로 자연스럽게 넘어가지 못하는 문제가 확인되었으므로, 우선순위를 다시 조정한다.

이번 개정에서 가장 시급한 문제는 다음이다.

1. 지역 미션 클리어 후 **다음 미션으로 자동 이동되지 않음**
2. “현재 미션”을 클리어했는지, 다음 지역으로 넘어갔는지 **상태가 불명확함**
3. “우리 주변” 클리어 후 “땅”으로 넘어갔지만 **지렁이만 표시됨**
4. 지역 미션 완료 후에도 **“현재 미션으로 돌아가기”만 활성화됨**
5. 지역 미션 창 진입 시 **사이드 로딩/목록 글씨가 갈색 계열로 겹쳐 보이지 않음**
6. 체크리스트가 상단에 있어 학생이 설명을 읽기 전 **무작정 클릭하고 넘어갈 가능성**이 있음

따라서 현재 단계의 개선 방향은 **기능 추가**가 아니라,
**지역 미션 진행 흐름을 수업 중 끊기지 않게 안정화하는 것**이다.

### 진행 현황

2026-04-30 기준 `60bc691 fix: advance mission flow after regional completion` 커밋에서 P0 지역 미션 전환 흐름을 반영했다.
이후 교사용 최초 실행 흐름을 `지역별 동물 범위 설정` 중심으로 바꾸고, 학생이 미션을 완료하지 않아도 사이드메뉴에서 다른 지역을 누르면 해당 지역에 대해 교사가 설정한 동물 카드가 보이도록 보강했다.
또한 설정 진입 문구를 `교사용 설정`으로 통일하고, 설정창에서 `지역 미션 설정`을 `AI 질문방 설정`보다 먼저 보이게 했다. 두 설정은 각각 따로 할 수 있음을 안내하며, 학생용 링크를 다음 기준으로 분기한다. 아무 설정도 없으면 `no-question.html` 기본 40마리 링크, 지역별 동물 범위만 설정하면 `no-question.html`에 지역 파라미터를 붙인 링크, AI 질문방만 설정하면 기본 40마리 `index.html?questionUrl=...` 링크, 둘 다 설정하면 지역 파라미터와 `questionUrl`이 함께 들어간 `index.html` 링크를 사용한다.
교사용 설정에는 교사용 기본 배포 주소와 학생용 링크/QR이 다르다는 설명을 추가했고, 공유 패널은 현재 상태에 따라 `기본 학생용 링크/QR` 또는 `설정 반영 링크/QR`로 제목과 설명을 바꾼다. AI 질문방을 연결하지 않는 선택은 `질문방 지우기`가 아니라 `AI 질문방 사용 안 함`으로 표시하여, 처음부터 질문방 없이 쓰는 경우와 이미 저장된 질문방을 해제하는 경우를 같은 선택으로 이해할 수 있게 했다.
배포 확인 중 HTML은 갱신됐지만 기존 `app.js` 캐시가 남아 공유 패널의 동적 문구가 늦게 바뀌는 상황을 확인했다. 이를 막기 위해 로컬 `styles.css`, `app-config.js`, `vendor/qrcode.js`, `app.js` 참조에 배포 버전 쿼리를 붙였다.
P0 체크 흐름은 한 덩어리 체크리스트가 아니라 `생김새`, `움직임`, `사는 곳` 설명 바로 아래에서 각각 확인하도록 분리했다. 학생은 관찰 설명을 읽는 자리에서 항목별로 체크한 뒤 퀴즈를 시작한다.
모바일 세로 퀴즈창에서는 진행바/음소거 HUD는 우상단에 유지하고, 상세/퀴즈 모달의 `← 뒤로가기` 버튼을 좌상단에 배치해 서로 겹치지 않게 했다. `관찰 요약 열기`는 상세 설명 하단이 아니라 퀴즈 문항 바로 아래에 접힘 패널로 배치하여, 학생이 모를 때 문항 아래에서 열고 닫을 수 있다.
P1 수업 UX는 미션 패널에 `지금 할 일` 문구를 추가하고, 학생용 첫 CTA를 `동물 관찰 시작하기`로 단순화했으며, 오답 시 노란 단서를 다시 보도록 안내한다. 교사용 설정에는 기본 주소 오공유 방지 문구와 `수업 전 5분 확인` 체크리스트를 추가했다.
P2는 실기기 검증을 제외하고 favicon, meta description, Open Graph/Twitter 공유 메타, Tailwind CDN 제거, NanumSquareRound CDN 제거, 구형 iPad용 `inert` 대체 처리, 모션 감소 대응, 로컬 이미지 manifest/detail 경로 검증을 반영했다.

검증: `node --check app.js`, 전체 계약 테스트, `npm run build:distribution` 기준으로 확인한다.

남은 확인 항목은 태블릿·전자칠판 실기기 최종 QA와 카카오톡/문자/클래스팅 실제 공유 캐시 확인이다.

# 3. P0 긴급 개선 항목

## P0-1. 지역 클리어 후 “다음 미션 시작” 동작 수정

### 실제 피드백

> 지역 클리어 후 “다음 미션 시작”을 눌러도 다음 지역 목록으로 자동 이동되지 않음.

### 문제 해석

학생이 한 지역 미션을 완료했는데도 다음 지역 카드 목록으로 전환되지 않으면,
교사는 수업 중 “다음으로 넘어가세요”라고 안내했음에도 학생 화면이 이전 상태에 머물 수 있다.

초등학교 3학년 수업에서는 이 문제가 단순한 불편을 넘어
수업 흐름 중단으로 이어질 가능성이 크다.

### 개선 방향

지역 미션 완료 후 `다음 미션 시작` 버튼을 누르면 다음 동작이 한 번에 일어나야 한다.

1. 현재 지역을 완료 상태로 저장
2. 다음 지역을 현재 미션으로 설정
3. 동물 카드 목록을 다음 지역 기준으로 필터링
4. 화면을 다음 지역 미션 또는 카드 목록 위치로 자동 스크롤
5. 상단/미션 패널에 “다음 미션으로 이동했습니다” 피드백 표시

### 권장 동작

```text
우리 주변 미션 완료
→ 다음 미션 시작 클릭
→ 현재 미션: 땅
→ 땅 지역 동물 카드 목록 표시
→ “땅 미션을 시작했어요” 안내 모달 또는 토스트 표시
```

### 완료 기준

- [x] `다음 미션 시작` 클릭 시 다음 지역 이름이 즉시 표시된다.
- [x] 다음 지역에 해당하는 동물 카드가 2개 이상 정상 표시된다.
- [x] 이전 지역 완료 상태가 유지된다.
- [x] 새로고침 후에도 현재 미션이 다음 지역으로 유지된다.
- [x] 학생이 추가 설명 없이 다음 카드를 누를 수 있다.

## P0-2. “현재 미션” 클리어 후 다음 지역 이동 상태 명확화

### 실제 피드백

> “현재 미션”을 클리어하고 다음 미션으로 제대로 이동하는지 여부가 불명확함.
> 교사가 미션을 주었더라도 학생이 clear했다면 다음 지역으로 넘어갈 수 있어야 함.

### 문제 해석

현재 UX에서는 다음 상태가 명확히 구분되지 않는 것으로 보인다.

- 현재 미션 진행 중
- 현재 미션 완료
- 다음 미션 대기
- 다음 미션 시작 완료

이 네 상태가 화면에서 분명히 구분되지 않으면 학생은 무엇을 해야 할지 모르고,
교사는 학생 화면을 일일이 확인해야 한다.

### 개선 방향

지역 미션 상태를 명확히 네 단계로 구분한다.

| 상태 | 화면 문구 | 버튼 |
|---|---|---|
| 진행 전 | `이번 미션: 우리 주변 동물 관찰하기` | `미션 시작하기` |
| 진행 중 | `우리 주변 미션 진행 중` | `현재 미션으로 돌아가기` |
| 완료 | `우리 주변 미션 완료!` | `다음 미션 시작` |
| 다음 미션 이동 후 | `현재 미션: 땅` | `땅 미션 시작하기` 또는 `현재 미션으로 돌아가기` |

### 문구 예시

```text
우리 주변 미션을 완료했어요!
이제 다음 미션인 “땅에 사는 동물”로 이동할 수 있어요.
```

```text
땅 미션을 시작했어요.
아래 카드에서 땅에 사는 동물을 관찰해 보세요.
```

### 완료 기준

- [x] 완료된 지역과 현재 진행 지역이 동시에 구분되어 표시된다.
- [x] 완료 후에는 `현재 미션으로 돌아가기`보다 `다음 미션 시작`이 우선 버튼으로 보인다.
- [x] 다음 미션 시작 후 현재 미션명이 새 지역으로 바뀐다.
- [x] 교사가 보지 않아도 학생이 다음 행동을 이해할 수 있다.

## P0-3. “우리 주변 → 땅” 이동 후 지렁이만 뜨는 문제 수정

### 실제 피드백

> “우리 주변” clear 후 “땅”으로 넘어갔는데 “지렁이”만 뜨고 있음.

### 문제 해석

이 문제는 단순 표시 오류가 아니라, 지역 미션 필터링 로직이나 동물 데이터 태그 문제일 가능성이 있다.

Possible causes are:
1. `땅` 지역에 속한 동물 데이터가 지렁이만 태깅되어 있음
2. 다음 지역 이동 시 필터 조건이 이전 상태와 충돌함
3. 지역 미션과 일반 도감 필터가 동시에 적용되어 결과가 과도하게 좁아짐
4. 미션 완료 후 카드 목록 갱신 함수가 일부 데이터만 렌더링함
5. 검색어/분류 필터/수집 상태 필터가 초기화되지 않아 지렁이만 남음

### 개선 방향

“다음 미션 시작” 시 카드 목록을 새 지역 기준으로 다시 그릴 때,
불필요한 필터를 초기화해야 한다.

특히 다음 값이 남아 있지 않은지 확인한다.

- 검색어
- 이전 지역 필터
- 수집 여부 필터
- 퀴즈 완료 여부 필터
- 현재 카드 인덱스
- 스크롤 위치
- 이전 미션의 selected animal state

### 권장 처리

```text
다음 지역 이동 시:
1. searchQuery = ""
2. selectedHabitatFilter = nextRegion
3. selectedAnimal = null
4. cardList = animals.filter(animal => animal.region === nextRegion)
5. renderCards(cardList)
6. scrollTo(cardSection)
```

### 완료 기준

- [x] “우리 주변” 완료 후 “땅”으로 이동하면 땅 지역 동물이 지렁이만 나오지 않는다.
- [x] 땅 지역에 속하는 모든 의도된 동물이 표시된다.
- [x] 검색어가 남아 있어 목록이 줄어드는 일이 없다.
- [x] 이전 지역 필터 and 다음 지역 필터가 중복 적용되지 않는다.
- [x] 각 지역별 표시 동물 수를 개발자가 확인할 수 있다.

### 추가 권장

디버그용으로 개발 모드에서 아래 정보를 표시하거나 콘솔에 출력한다.


## P0-4. 지역 미션 완료 후 버튼 활성화 로직 수정

### 실제 피드백

> 지역 미션 완료 후에도 “현재미션으로 돌아가기” 미션 버튼만 활성화되고 있음.

### 문제 해석

미션 완료 상태인데도 `현재 미션으로 돌아가기`가 주 버튼으로 남아 있으면,
학생은 이미 끝낸 미션으로 다시 돌아가게 된다.

완료 상태에서는 버튼 우선순위가 바뀌어야 한다.

### 개선 방향

버튼 활성화 조건을 미션 상태에 따라 분리한다.

| 조건 | 주 버튼 | 보조 버튼 |
|---|---|---|
| 미션 진행 전 | `미션 시작하기` | 없음 |
| 미션 진행 중 | `현재 미션으로 돌아가기` | `전체 도감 보기` |
| 미션 완료, 다음 지역 있음 | `다음 미션 시작` | `완료한 미션 다시 보기` |
| 미션 완료, 다음 지역 없음 | `전체 미션 완료` | `도감 보기` |

### 완료 기준

- [x] 미션 완료 직후 `다음 미션 시작` 버튼이 가장 크게 보인다.
- [x] `현재 미션으로 돌아가기`는 보조 버튼으로 내려간다.
- [x] 마지막 지역 완료 시에는 `전체 미션 완료` 상태가 표시된다.
- [x] 다음 지역이 없을 때 빈 버튼이나 잘못된 이동이 발생하지 않는다.

## P0-5. 지역 미션 창의 갈색 계열 글자 대비 문제 수정

### 실제 피드백

> 지역미션 창의 사이드로딩창의 글씨: 대기상태는 문제 없지만 진입했을 때 둘 다 갈색계열로 안 보임.

### 문제 해석

대기 상태에서는 보이지만 미션 창에 진입했을 때 보이지 않는다면,
활성 상태의 배경색과 글자색이 모두 갈색 계열로 변하거나,
선택/hover/active 클래스가 잘못 적용되었을 가능성이 있다.

이 문제는 수업 현장에서 매우 치명적이다.
학생이 현재 지역명, 미션 상태, 버튼 문구를 읽지 못하면 교사 안내가 반복될 수밖에 없다.

### 개선 방향

미션 창과 사이드 목록의 상태별 색상을 분리한다.

| 상태 | 배경 | 글자 |
|---|---|---|
| 대기 | 밝은 베이지/연한 숲색 | 진한 갈색 또는 진한 초록 |
| 활성 | 진한 초록 또는 남색 계열 | 흰색 |
| 완료 | 연한 초록 | 진한 초록 |
| 비활성 | 연한 회색 | 중간 회색 |

### 접근성 기준

- 일반 텍스트: 명도 대비 4.5:1 이상
- 큰 글자/버튼 텍스트: 명도 대비 3:1 이상
- 색상만으로 상태를 구분하지 않고 아이콘/문구 병행

### 문구 예시

```text
진행 중
완료
다음 미션
잠김
```

### 완료 기준

- [x] 미션 창 진입 후에도 모든 지역명과 상태 문구가 읽힌다.
- [x] 활성 지역은 배경과 글자 대비가 명확하다.
- [x] 완료 지역과 현재 지역이 색상뿐 아니라 문구로도 구분된다.
- [ ] 태블릿 밝기 50%에서도 확인 가능하다.
- [ ] 전자칠판 화면에서도 지역명 식별이 가능하다.

## P0-6. 관찰 체크를 설명 항목별로 분리

### 실제 피드백

> 관찰 체크창은 한번에 3가지를 몰아넣지 않고, 각 설명 항목 밑에 따로따로 체크하게 해줘.

### 문제 해석

세 가지 체크가 한 박스에 몰려 있으면 학생은 설명을 다 읽기 전에 체크만 빠르게 누를 수 있다.
체크박스는 “넘어가기 버튼”이 아니라 바로 위 설명을 읽고 확인했다는 표시가 되어야 한다.

### 개선 방향

동물 상세 설명 구조를 다음 순서로 바꾼다.

1. 생김새 설명
2. `생김새를 봤어요` 체크
3. 움직임 설명
4. `움직이는 방법을 봤어요` 체크
5. 사는 곳 설명
6. `사는 곳을 봤어요` 체크
7. 퀴즈 시작 버튼

### 권장 화면 흐름

```text
생김새와 움직임

무당벌레의 몸에서는 단단한 겉날개, 작은 다리, 점무늬 같은 부분이 잘 보입니다.
[ ] 생김새를 봤어요

다리로 걷고 날개로 날아가요.
[ ] 움직이는 방법을 봤어요

사는 곳과 생활

풀밭, 나뭇잎에서 먹이나 쉴 곳을 찾으며 살아갑니다.
[ ] 사는 곳을 봤어요

[동물 관찰 시작하기]
```

### 완료 기준

- [x] 세 가지 관찰 체크가 하나의 박스에 몰려 있지 않다.
- [x] 각 체크는 관련 설명 문장 바로 아래에 있다.
- [x] 세 가지 체크 전에는 퀴즈 시작 버튼이 비활성화된다.
- [x] 학생이 설명을 읽는 흐름 안에서 체크하도록 구성된다.

## P0-7. 모바일 세로 퀴즈창 뒤로가기와 관찰 요약 위치 수정

### 실제 피드백

> 모바일의 세로 방향 퀴즈창에서 뒤로가기 버튼과 카드 진행 모달창이 계속 겹쳐서 음소거 버튼만 누르고 실행이 안되는 중임.
> 이전에 상세 카드 퀴즈 모달 뒤로가기 위치를 좌상단으로 올렸었는데 반영이 안됨.
> 관찰 요약 열기는 설명창이 아니라 문제를 푸는 문항 바로 아래에 배치해서 모를때 학생이 열고 닫을 수 있게 해줘.

### 문제 해석

진행바와 음소거 버튼이 앱 HUD로 상단에 고정되는 것은 정상 동작이다.
문제는 퀴즈/상세 모달의 뒤로가기 버튼이 이 HUD와 같은 영역에서 경쟁하면서, 학생이 뒤로가기를 누르려다 음소거 버튼을 누르는 흐름이 생기는 것이다.

또한 관찰 요약이 상세 설명창 하단에 있으면, 실제 문제를 풀 때 학생이 힌트를 찾기 위해 다시 설명 화면을 오가야 한다.
요약은 문제 아래에서 필요할 때만 여는 보조 단서가 되어야 한다.

### 개선 방향

- 진행바/음소거 HUD는 우상단에 유지한다.
- 상세/퀴즈 모달의 `← 뒤로가기` 버튼은 좌상단에 고정한다.
- 모달 제목 sticky 영역이 뒤로가기 버튼을 덮지 않도록 버튼 z-index를 제목 영역보다 높게 유지한다.
- `관찰 요약 열기`는 퀴즈 문항 `<h3>` 바로 아래, 선택지 위에 배치한다.

### 완료 기준

- [x] 모바일 세로 퀴즈 화면에서 `← 뒤로가기`와 음소거 버튼이 좌우로 분리된다.
- [x] 진행바/음소거 HUD는 계속 보인다.
- [x] 퀴즈 문항 바로 아래에 `관찰 요약 열기`가 접힘 상태로 표시된다.
- [x] 학생은 문제를 보다가 모를 때 요약을 열고 닫을 수 있다.

# 4. 지역 미션 상태 설계 재정리

현재 피드백의 핵심은 지역 미션의 상태 전환이 불명확하다는 점이다.
따라서 상태 모델을 명확히 정리해야 한다.

## 4.1 권장 상태값

```text
notStarted   = 아직 시작하지 않음
inProgress   = 진행 중
completed    = 완료됨
nextReady    = 다음 미션 시작 가능
allCompleted = 전체 완료
```

## 4.2 상태 전환 흐름

```text
미션 시작하기 클릭
notStarted → inProgress

지역 동물 조건 충족
inProgress → completed

다음 미션 시작 클릭
completed → nextReady → 다음 지역 inProgress

마지막 지역 완료
completed → allCompleted
```

## 4.3 화면별 우선 표시

| 화면 | 우선 표시해야 할 정보 |
|---|---|
| 첫 진입 | 오늘의 현재 미션 |
| 진행 중 | 현재 미션명, 남은 동물 수 |
| 완료 직후 | 완료 축하, 다음 미션 버튼 |
| 다음 이동 직후 | 새 지역명, 관찰할 동물 목록 |
| 전체 완료 | 전체 완료 축하, 도감 보기 |

# 6. 구현 시 주의할 점

## 6.1 단순히 버튼만 추가하면 안 됨

이번 문제는 버튼 문구 문제가 아니라 상태 전환 문제다.
따라서 `다음 미션 시작` 버튼을 보이게 하는 것만으로는 부족하다.

반드시 다음이 함께 처리되어야 한다.

- 현재 지역 상태 저장
- 다음 지역 계산
- 다음 지역 상태 활성화
- 카드 목록 필터 초기화
- 카드 목록 재렌더링
- 화면 이동 또는 안내 모달 표시
- 새로고침 후 상태 복원

## 6.2 필터 중복 적용 가능성 확인

“땅” 지역에서 지렁이만 뜨는 문제는 필터 중복 가능성이 높다.
지역 이동 시 기존 검색어, 이전 필터, 수집 상태 필터가 남아 있지 않은지 확인해야 한다.

## 6.3 학생용 문구를 먼저 설계해야 함

초3 학생은 “상태 전환”을 추상적으로 이해하지 않는다.
따라서 내부 상태보다 화면 문구가 중요하다.

나쁜 문구:

```text
현재 미션으로 돌아가기
```

상황별로 더 나은 문구:

```text
우리 주변 미션을 완료했어요
다음 미션: 땅에 사는 동물
땅 미션 시작하기
땅 동물 카드를 관찰해 보세요
```

## 6.4 교사 개입을 줄이는 방향으로 설계

수업 중 교사가 계속 “다음 지역 눌러”, “뒤로 가”, “그거 말고 이거 눌러”라고 말해야 한다면 UX가 실패한 것이다.
학생이 화면만 보고 다음 행동을 알 수 있어야 한다.

# 7. 수업 전 QA 시나리오

## 7.1 지역 미션 기본 흐름 테스트

```text
1. 새 브라우저 또는 수업 기록 초기화 상태에서 접속
2. 첫 지역 미션 시작
3. 해당 지역 동물 카드 관찰
4. 퀴즈 완료
5. 지역 미션 클리어 확인
6. 다음 미션 시작 클릭
7. 다음 지역명 표시 확인
8. 다음 지역 동물 카드 목록 확인
9. 새로고침
10. 현재 미션이 다음 지역으로 유지되는지 확인
```

## 7.2 “우리 주변 → 땅” 집중 테스트

```text
1. 우리 주변 미션 시작
2. 우리 주변 미션 완료
3. 다음 미션 시작 클릭
4. 현재 미션이 땅으로 바뀌는지 확인
5. 땅 지역 동물 카드가 지렁이만 뜨지 않는지 확인
6. 검색창, 필터, 수집 상태가 초기화되었는지 확인
```

## 7.3 버튼 상태 테스트

```text
진행 전: 미션 시작하기
진행 중: 현재 미션으로 돌아가기
완료 후: 다음 미션 시작
마지막 완료 후: 전체 미션 완료
```

## 7.4 가독성 테스트

```text
1. 지역 미션 창 대기 상태 확인
2. 지역 미션 창 진입 상태 확인
3. 활성 지역명 읽기
4. 완료 지역명 읽기
5. 태블릿 밝기 50%에서 확인
6. 전자칠판 전체 화면에서 확인
```

## 7.5 관찰 체크와 퀴즈 요약 흐름 테스트

```text
1. 동물 상세창 열기
2. 생김새/움직임/사는 곳 설명이 먼저 보이는지 확인
3. 각 설명 항목 바로 아래에 해당 관찰 체크가 있는지 확인
4. 세 가지 체크 전에는 퀴즈 시작 버튼이 비활성화되는지 확인
5. 퀴즈 화면에서 문항 바로 아래 `관찰 요약 열기`가 보이는지 확인
6. 모바일 세로 화면에서 `← 뒤로가기`와 음소거 버튼이 겹치지 않는지 확인
```

# 9. 최종 결론

이번 피드백은 단순한 디자인 개선이 아니라 **수업 진행을 막는 흐름 오류**에 가깝다.
따라서 기존 로드맵에서 장기 과제로 두었던 모바일, meta, PWA, 문서 정리보다 우선하여 다음을 먼저 처리해야 한다.

1. 지역 미션 완료 후 다음 지역으로 확실히 이동
2. 다음 지역 카드 목록이 정확히 표시
3. 완료 후 버튼 우선순위 수정
4. 현재/완료/다음 미션 상태를 학생 눈높이에 맞게 표시
5. 미션 창 글자 대비 수정
6. 관찰 체크를 설명 항목별로 분리하고 퀴즈 문항 아래에 관찰 요약 배치

이 여섯 가지가 해결되어야 실제 수업 중 교사가 개입하지 않아도 학생이 자연스럽게 다음 미션을 이어갈 수 있다.

따라서 최신 로드맵의 첫 번째 목표는 다음과 같이 정리한다.

> **목표: 학생이 지역 미션을 클리어한 뒤, 교사의 추가 설명 없이 다음 지역 미션으로 자연스럽게 이동할 수 있게 한다.**
