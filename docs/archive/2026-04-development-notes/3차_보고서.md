## 6. 기술적 부채 및 유의사항

### 6.1 `no-question.html` 동기화
- 현재 `index.html`을 source of truth로 두고 `no-question.html`을 **자동 생성**하도록 전환됨
- 질문방 관련 UI/스크립트 제거 여부는 `scripts/phase3-contract.test.mjs`로 계약 테스트 검증
- **유지 원칙**: 학생용 페이지 구조 변경은 `index.html`과 생성기 marker 규칙을 함께 검토

### 6.2 이미지 내장 현재 상태
- `scripts/download-local-images.js`가 `images/thumbs/`, `images/details/`, `images/manifest.json`을 생성
- `scripts/build.js`는 local image manifest를 읽어 `dist/app-config.js`에 경로를 주입
- `app.js`는 manifest 기반 local image → Wikimedia/source → SVG placeholder 순서로 폴백
- **유의**: 현재 이미지 번들은 약 5MB로 목표치보다 커서 추가 압축 여지가 있음

### 6.3 효과음 브라우저 호환성
- Web Audio API는 iOS Safari에서 **사용자 제스처 이후**에만 초기화 가능
- 현재 `ensureAudioContext()`는 `playSound()` 호출 시마다 확인하므로, 첫 터치 이후 정상 작동
- **유의**: 온보딩 "시작하기" 버튼이 첫 제스처 역할을 하므로 문제 없음

### 6.4 용량 현황
- 코드(`app.js` + `styles.css` + HTML): ~150KB
- 효과음: 0KB (Oscillator 합성음, 파일 없음)
- 이미지(현재): 약 5MB (`images/` 기준, 54마리 전체 local bundle)
- **총 배포 현황**: `dist/images` 약 5MB. 학교망 안정성은 확보했지만 추가 압축 여지 있음

## 7. 종합 평가

**PR #8 (`codex/featurerefactoring`)은 2차 개선계획의 핵심 항목 대부분을 훌륭하게 구현했습니다.**

- **학습 동기**: 지역별 마일스톤 + 보상 모달로 54마리라는 거대한 목표를 4단계로 세분화하여 지속적인 성취감 제공
- **사용성**: 온보딩 튜토리얼 + 상단 고정 진행도(`stickyProgress`) + 안전 약속 접기(`<details>`)로 시작 화면 복잡도 해소
- **몰입도**: Web Audio API Oscillator 합성음으로 외부 파일 의존 없이 효과음 구현 (용량 0 증가, 정적 사이트 제약 극복)
- **안정성**: 검색 디바운스 + `safeSetStorage()`로 localStorage 안정성 강화

**가장 큰 구조 작업이었던 이미지 내장(3.6)과 `no-question.html` 자동 생성(3.7.3)은 완료되었습니다.** 현재 배포본은 로컬 이미지 자산을 우선 사용하고, `credits.html`로 사진 출처를 일괄 확인할 수 있으며, 학생용/교사용 HTML도 자동 동기화됩니다.

이제 남은 일은 구현보다 **현장 검증과 경량화**에 가깝습니다. 게임 모바일 실기기 QA, 전자칠판 수업 흐름 확인, 그리고 필요 시 이미지 번들 추가 압축을 진행하면 실제 수업 투입 안정성이 한 단계 더 올라갑니다.
