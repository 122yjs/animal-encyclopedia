# 🐾 부엉이 동물도감 탐험대 UX 평가 및 개선 계획 (통합 업데이트)

본 문서는 **'동물도감 54 통합 개발계획서'** 및 **'3차 개발 진척상황 보고서'**를 바탕으로 현재 프로젝트의 사용자 경험(UX) 관점 장단점을 분석하고, 실제 교실 투입을 위한 구체적인 개선 로드맵을 정리한 문서입니다.

현재 프로젝트는 **Phase 0 ~ Phase 3까지의 주요 기능(이미지 내장, 지역별 미션, `no-question.html` 자동화 등)이 약 91% 구현 완료**된 상태로, 기능 추가보다는 **학생 화면 단순화와 실기기 안정화(QA)**에 초점이 맞춰져 있습니다.

---

## 1. 현재 상태의 UX 장단점 평가

### 👍 장점 (Pros)
1. **수업에 최적화된 게임화(Gamification) 구조**
   - 54마리 전체를 한 번에 수집하는 부담을 줄이고, **'지역별 미션(6~8마리)'과 '진행도에 따른 마일스톤 모달 보상'**으로 세분화하여 초등학교 3학년 학생의 40분 수업 집중도를 유지합니다.
2. **학생과 교사를 위한 듀얼 페르소나 (Dual Persona UX)**
   - `index.html`과 `no-question.html`을 분리하여 학생 화면에서는 불필요한 설정 문구를 덜어냈습니다. 교사는 오늘의 미션 지역을 설정하고, 학생은 링크 하나로 온전히 관찰과 퀴즈에 몰입할 수 있습니다.
3. **환경 제약을 극복하는 안정성(Offline / Network)**
   - **이미지 로컬 내장(Fallback: Local → Wikimedia → Placeholder)**을 통해 느린 학교망에서도 카드가 안정적으로 렌더링됩니다.
   - `safeSetStorage()`를 통해 로컬 스토리지 에러를 방지하고, Web Audio API 합성음(Oscillator)으로 외부 음원 파일(mp3) 의존성 없이 경쾌한 효과음을 제공합니다.
4. **학생의 인지적 부담(읽기·조작) 완화**
   - 상세 설명 하단에 **'관찰 요약(Quick Facts)' 접힘 패널**을 적용하고, 퀴즈 오답 시 **본문 노란색 하이라이트**로 힌트를 주어 읽기 부담을 줄였습니다.
   - 3단계 튜토리얼(온보딩) 지원, 검색어 칩(예: 날개, 지느러미 등) 적용 등 스스로 탐색하기 좋은 UX를 갖추고 있습니다.

### 👎 단점 및 남아있는 과제 (Cons & Pain Points)
1. **분류 게임의 모바일 조작의 한계 (모바일 UX)**
   - 스마트폰이나 태블릿(Galaxy Tab A8 등) 세로 모드에서 상하 스크롤이 길고, 카드를 직접 드래그 앤 드롭해야 하는 인터랙션이 아직 매끄럽지 않은 부분이 있습니다.
2. **과도한 에셋(이미지) 용량**
   - 로컬 이미지 내장화는 달성했으나, 현재 54종 전체의 번들 용량이 약 5MB에 달하여 권장 목표치(1.5~2MB)에 비해 로딩 최적화 여지가 남아 있습니다.
3. **구형 태블릿 호환성 이슈 우려**
   - 모달 뒤 레이어의 접근성을 제어하는 `inert` 속성이 구형 iPad(iOS 14 이하)에서는 작동하지 않을 수 있어, 화면 조작 시 의도치 않은 버그가 발생할 수 있습니다.
4. **신규 테마(V2 숲 테마)의 시각적 안정성 점검**
   - 다크 우드·골드 톤을 적용한 V2 리디자인이 시각적으로 훌륭하나, 저해상도 화면에서의 글자 가독성이나 Tailwind CDN 차단 시의 대응 등에 대한 꼼꼼한 QA가 대기 중입니다.

## 2. 향후 개선 계획 (Action Plan)

가장 큰 기술 부채였던 '이미지 번들링'과 'HTML 빌드 파이프라인 분리'가 완료되었습니다. 남은 작업은 새로운 기능 추가를 지양하고 **UX 디테일 및 태블릿 교실 환경 검증**을 최우선으로 진행합니다.

### 🚀 단기 과제: 분류 게임 UX 완성 및 실기기 검증 (Phase 0 ~ 1 잔여 작업)
- **모바일 분류 게임 2단 레이아웃 완성:** 모바일 세로 화면에서 드래그 없이 **"카드 탭 → 영역(맞아요/달라요) 탭"**만으로 직관적으로 분류할 수 있도록 코드를 최적화하고 실기기 테스트를 반복합니다. (또한 '카드 더미(pool)'로 손쉽게 되돌리는 로직 확립)
- **전자칠판 & 태블릿 연계 QA:** 수업 시나리오를 바탕으로 교사 PC(전자칠판)로 지역 미션을 생성 및 초기화(리셋 버튼)하고, 학생 태블릿에서 오류 없이 진행되는지 통합 테스트를 수행합니다.

### 🌐 중장기 과제: 용량 최적화 및 브라우저 호환성 확보
- **이미지 용량 추가 압축:** `images/thumbs/`와 `images/details/` 내 파일들을 압축 알고리즘(예: WebP, 압축률 조정)을 적용하여 전체 용량을 2MB 이하로 줄여 쾌적한 로딩을 보장합니다.
- **구형 기기 폴리필(Polyfill) 도입:** 구형 브라우저(iOS 14 이하) 환경을 타겟팅해야 하는 경우를 대비해 `inert` 폴리필을 추가 적용하여, 키보드 포커스와 터치 접근성 결함을 메웁니다.
- **V2 테마 리디자인 접근성(A11y) 검수:** 브랜치 병합 전 가독성(대비율), 감소된 동작 선호 시의 애니메이션 속도, Tailwind 의존성 이탈 시의 기본 레이아웃 유지 여부를 면밀히 확인합니다.

'부엉이 동물도감 탐험대'는 단순 자료 제공 사이트를 넘어 초등 3학년 수준에 맞춘 **'완성도 높은 교육용 웹앱'**으로 발전했습니다. 앞으로 **태블릿 터치 환경 최적화(분류 게임 등), 이미지 경량화, 구형 기기 호환성 대비** 등의 백로그를 해소한다면 일선 교실에 즉각적이고 안정적으로 투입할 수 있을 것입니다.
