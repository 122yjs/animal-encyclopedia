## 1. 프로젝트 한 줄 정의

**동물도감 54**는 초등학교 3학년 학생이 54종 동물의 생김새, 사는 곳, 이동 방식, 환경 적응 특징을 관찰하고, 퀴즈와 분류 미션을 통해 동물의 생활 개념을 익히는 **게임형 과학 수업 웹앱**이다.

단순한 동물 정보 사이트가 아니라 다음 흐름을 가진 수업 도구로 보는 것이 적절하다.

```text
동물 카드 선택
→ 사진과 핵심 특징 관찰
→ 퀴즈 풀이
→ 도감 등록
→ 지역별/묶음별 수집
→ 분류 미션으로 개념 정리
→ 필요 시 AI 질문방 또는 질문방 없는 수업 모드 운영
```

## 3. 현재까지 확인된 강점

### 3.1 게임화 구조가 명확하다

현재 프로젝트의 가장 큰 강점은 **도감 등록, 진행도, 포획 애니메이션, 지역 완성 보상**이 결합되어 있다는 점이다.  
초등학교 3학년 학생에게 “조사하기”를 단순 읽기 과제로 제시하면 집중 시간이 짧아질 수 있는데, 이 프로젝트는 동물 카드를 수집하는 방식으로 학습 동기를 유지한다.

보존해야 할 요소는 다음과 같다.

- 퀴즈 통과 후 도감에 등록되는 성취감
- 등록 배지와 수집 진행도
- 지역별 마일스톤과 보상 모달
- 전체 완성 시 도감 마스터 보상
- 학생 친화적인 색감, 카드형 UI, 애니메이션

### 3.2 퀴즈-힌트-재도전 구조가 교육적으로 좋다

오답을 단순 실패로 처리하지 않고, 관련 문단을 다시 보게 하거나 재도전하도록 유도하는 구조는 과학 수업에서 좋다.

교실에서는 정답을 맞히는 것보다 **관찰 근거를 다시 찾는 과정**이 중요하다.  
따라서 퀴즈 구조는 유지하되, 다음과 같은 보강이 필요하다.

- 퀴즈 전 “관찰 체크” 삽입
- 정답 후 “어떤 단서를 보고 알았나요?” 문장 제시
- 오답 후 “다시 볼 곳”을 더 명확하게 표시

### 3.3 교사용/학생용 흐름 분리가 이미 되어 있다

`index.html`은 질문방 설정이 가능한 교사용 흐름이고, `no-question.html`은 질문방 기능을 제거한 학생용 흐름으로 구분되어 있다.  
이 점은 매우 중요하다.

다만 질문방 없는 버전에서도 학생에게 불필요한 “질문방 설정” 계열 문구가 일부 보이면 혼란을 줄 수 있으므로, 향후 개선은 **기능 추가가 아니라 학생 화면에서 교사용 문구를 덜어내는 방향**이 적절하다.

### 3.4 접근성·정적 사이트 구조가 비교적 안정적이다

현재 프로젝트는 정적 HTML/CSS/JS 기반이며, 별도의 복잡한 런타임 의존성이 적다.  
또한 모달 포커스 처리, `aria-live`, 감소된 동작 선호 설정, lazy loading 등 접근성·성능 요소를 고려한 흔적이 있다.

이 구조는 학교 현장에서 유리하다.

- 설치 없이 링크 접속 가능
- GitHub Pages 배포 가능
- 공용 태블릿에서 비교적 단순하게 사용 가능
- 전자칠판과 학생 태블릿을 함께 활용하기 쉬움

## 4. 사용자 UX 관점 핵심 페인포인트

### 4.1 첫 화면 정보량이 많다

현재 첫 화면은 기능이 풍부한 만큼 초등 3학년에게는 복잡하게 느껴질 수 있다.

학생 입장에서는 다음 질문이 생길 수 있다.

- “무엇부터 눌러야 하지?”
- “도감 모드와 분류 미션은 뭐가 다르지?”
- “54마리를 다 해야 하나?”
- “질문방 설정은 내가 해야 하나?”

따라서 첫 화면은 학생용으로 더 단순해야 한다.

권장 방향:

```text
오늘의 미션
1. 동물 카드 3개를 관찰해요.
2. 퀴즈를 풀어 도감에 등록해요.
3. 친구와 동물을 나누는 기준을 말해요.

[동물 관찰 시작하기]
[카드 나누기 게임하기]
[내 도감 보기]
```

### 4.2 54마리 전체 목표는 부담이 될 수 있다

54마리 전체 도감은 장기 과제로는 좋지만, 40분 수업에서는 과하다.  
지역별 마일스톤이 구현되었더라도, 수업 차시에서는 **교사가 오늘 진행할 지역과 동물 수를 제한하는 장치**가 필요하다. 이때 학생 화면에는 별도의 “오늘 수업” 보상이 아니라, 선택된 지역 미션의 일부로 보여야 한다.

권장 방향:

| 모드 | 동물 수 | 용도 |
|---|---:|---|
| 교사용 지역 미션 설정 | 6~8마리 권장 | 1차시 수업용 |
| 지역 미션 모드 | 지역별 선택 동물 수 | 2~3차시 확장 |
| 전체 도감 모드 | 54마리 | 자유 탐구·가정 연계 |
| 교사용 추천 세트 | 교사가 선택 | 공개수업·수행평가용 |

### 4.3 동물 상세 설명은 읽기 흐름을 해치지 않게 보조 단서를 제공해야 한다

상세 설명은 교육적으로 풍부하지만, 3학년 학생에게는 읽기 부담이 생길 수 있다.  
초기 화면에서 별도 `관찰 단서` 카드가 먼저 나오면 설명 본문, 퀴즈 힌트, 요약 카드가 서로 경쟁할 수 있다. 따라서 현재 구현에서는 상세 설명을 먼저 읽고, 필요할 때 하단의 접힘 패널에서 핵심 요약을 확인하는 방식으로 조정했다.

현재 구성:

```text
설명 본문

[관찰 요약 열기]
- 사는 곳
- 움직임
- 특징

[퀴즈 풀고 도감에 등록하기]
```

오답 후에는 이 요약 패널이 자동으로 튀어나오지 않고, 본문 안의 관련 문장이 노란색으로 강조된다.  
즉, **선택형 요약은 하단 접힘 패널**, **오답 힌트는 본문 하이라이트**로 역할을 분리한다.

### 4.4 검색 기능은 예시어가 더 필요하다

3학년은 스스로 검색어를 떠올리는 데 어려움을 겪을 수 있다.  
검색창에는 이름 검색뿐 아니라 특징 검색이 가능하다는 것을 보여 주어야 한다.

권장 예시:

```text
동물 이름이나 특징을 찾아보세요.
예: 날개, 지느러미, 사막, 물속, 다리, 털
```

추천 검색어 칩:

```text
[날개] [다리] [지느러미] [물속] [사막] [땅속] [알] [털]
```

### 4.5 분류게임 모바일 조작이 가장 큰 남은 UX 과제다

분류게임은 과학 개념 형성에 매우 좋은 활동이지만, 모바일에서는 상하 스크롤이 길고 드래그 조작이 어려울 수 있다.

현재 개선 방향은 다음과 같이 확정하는 것이 좋다.

- 모바일에서는 카드 더미를 상단 가로 스크롤 줄로 배치
- 아래에는 `맞아요 / 달라요` 두 영역을 2열로 배치
- 카드를 끌지 않아도 “카드 탭 → 영역 탭”으로 이동 가능하게 함
- 이미 옮긴 카드를 다시 카드 더미로 되돌릴 수 있게 함
- `gamePool`에도 `drop-zone` 역할과 `data-answer="pool"`을 부여함

## 5. 개발 현황 통합 정리

### 2026-04-29 진행 업데이트

2026-04-29 현재 `codex/animal-roadmap-20260428` 브랜치 기준으로 Phase 0 일부와 Phase 1 핵심 흐름이 구현·검증되었다.  
중간 커밋은 `177bbf8`, `2ecd7f8`, `5405e54`까지 원격 브랜치에 푸시되어 있다.

이번 개발에서 확정된 UX 방향은 다음과 같다.

- “오늘 수업”은 별도 미션이 아니라 **교사가 설정하는 지역 미션**으로 통합한다.
- 보상 구조는 **지역별 보상 → 전체 보상**으로 일원화한다.
- 교사는 지역별로 등장할 동물 종류와 수를 설정하고, 학생은 공유 링크로 선택된 지역 미션만 수행한다.
- `전체 도감` 숫자는 전체 54마리 고정이 아니라, 교사가 선택한 지역별 미션 동물 집합에 맞춰 조정된다.
- `관찰 단서`는 초기 화면에서 별도 카드로 노출하지 않고, 상세 설명 하단의 접힘 패널인 **관찰 요약**으로 제공한다.
- 오답 후 힌트는 별도 패널이 아니라, 관련 본문 문단의 **노란 하이라이트**로 일원화한다.
- `퀴즈 전 관찰 체크`는 동물별 최초 1회 완료 후 사라지며, 이후 같은 동물을 다시 볼 때 반복 노출되지 않는다.
- `퀴즈 풀고 도감 등록` 버튼은 설명 본문과 관찰 요약 뒤에 배치해, 먼저 읽고 관찰한 뒤 퀴즈로 이동하도록 조정했다.
- 모바일 퀴즈/상세 모달에서는 닫기 버튼을 왼쪽 상단으로 옮기고, 포획 애니메이션과 보상 모달이 퀴즈 모달 뒤로 가려지지 않도록 레이어 순서를 복구했다.

검증은 다음 명령으로 수행했다.

```bash
node --test scripts/phase0-contract.test.mjs
node --test scripts/phase1-contract.test.mjs
node --check app.js
npm run build:distribution
git diff --check
```

## 6. 개발 원칙

### 6.1 기능을 더 넣기보다 학생 화면을 덜어낸다

현재 프로젝트는 이미 기능이 많다.  
따라서 단기 개선의 핵심은 기능 추가보다 **3학년 학생이 수업 중 헷갈리지 않도록 화면을 단순화하는 것**이다.

원칙:

- 학생에게 필요 없는 교사용 설정은 숨긴다.
- 첫 화면에는 오늘 해야 할 행동만 남긴다.
- 동물 상세 정보는 핵심 단서 → 자세히 보기 순서로 보여 준다.
- 외부 링크는 학생이 의도하지 않게 나가지 않도록 한 번 더 안내한다.

### 6.2 전체 도감보다 오늘의 지역 미션을 우선한다

도감 전체 완성은 좋은 장기 목표이지만, 수업 시간에는 차시 목표가 우선이다.

원칙:

- 교사가 오늘 진행할 지역과 동물 수/종류를 정할 수 있어야 한다.
- 학생은 6~8마리만 완성해도 해당 지역 미션 완성 경험을 얻어야 한다.
- 빠른 학생은 전체 도감으로 확장할 수 있게 한다.

### 6.3 모바일·태블릿 조작을 기본 기준으로 삼는다

실제 수업에서는 마우스보다 손가락 터치가 많다.  
따라서 개발 기준은 데스크톱이 아니라 공용 태블릿이어야 한다.

원칙:

- 드래그 앤 드롭만으로 기능을 설계하지 않는다.
- 항상 탭 기반 대체 조작을 제공한다.
- 버튼과 카드 터치 영역은 충분히 크게 둔다.
- 가로모드와 세로모드를 모두 확인한다.

### 6.4 질문방 없는 버전을 수업 기본값으로 둔다

AI 질문방은 심화 활동으로는 좋지만, 3학년 기본 수업에서는 외부 이동과 질문 분산의 위험이 있다.

원칙:

- 기본 수업 링크는 `no-question.html`을 우선 사용한다.
- AI 질문방은 교사가 의도한 차시에만 사용한다.
- 학생용 화면에서는 질문방 설정 문구를 숨긴다.
- 질문방 사용 시에도 돌아오기 안내를 명확히 제공한다.

### 6.5 공개 배포 가능성을 고려한다

학교 현장에서 공유할 가능성이 있으므로, 사진 출처와 라이선스 표기는 더 안정적으로 관리해야 한다.

원칙:

- 각 사진별 출처를 추적 가능하게 한다.
- “사진 출처 일괄 보기” 페이지를 만든다.
- 외부 이미지를 내장하더라도 원출처 기록은 유지한다.
- 특정 IP를 연상시키는 용어는 장기적으로 자체 용어로 교체한다.

## 8. 신규 기능 상세 설계

## 8.1 교사용 지역 미션 설정

### 목적

54마리 전체 도감을 40분 수업 목표로 제시하지 않고, 교사가 차시별 목표에 맞는 지역과 동물만 보여준다. 보상 구조는 “지역별 보상 → 전체 보상”으로 일원화한다.

### 권장 UI

```text
오늘의 지역 미션을 설정하세요.  ※ 교사용

[지역 선택: 특별한 환경]
[낙타] [도루묵도마뱀] [사막여우] [사막 뱀]
[사막 딱정벌레] [북극곰] [북극여우] [펭귄]
[학생용 링크 생성]
```

학생 화면:

```text
오늘의 지역 미션: 특별한 환경
0 / 8

[동물 관찰 시작하기]
```

### 구현 방식

- `state.missionRegion = "special"`
- `state.missionAnimalIds = ["낙타", "도루묵도마뱀", ...]`
- 카드 렌더링 시 선택된 지역/동물만 표시
- 공유 링크는 `?set=special&animals=...` 형태로 학생 화면에 전달
- 보상은 별도 “오늘 수업 보상”이 아니라 기존 지역 완성 보상을 사용

### 완료 기준

- URL 파라미터로 세트 공유 가능  
  예: `?set=special&animals=낙타,북극곰,펭귄`
- 학생은 선택된 지역 동물을 모두 등록하면 해당 지역 완성 보상을 받음
- 전체 도감 보상은 54마리 전체 완성 시에만 제공됨

## 8.2 관찰 요약 접힘 패널

### 목적

읽기 부담을 낮추되, 설명 본문과 오답 힌트가 서로 경쟁하지 않게 한다.  
초기 화면에는 본문 설명을 먼저 보여 주고, 학생이 필요할 때만 `관찰 요약`을 펼쳐 핵심 3가지를 확인하게 한다.

### 권장 UI

```text
[관찰 요약 열기]

사는 곳: 숲과 나무 위
움직임: 날거나 나뭇가지를 뛰어다님
특징: 날개와 깃털이 있음
```

### 구현 방식

동물 데이터 구조에 다음 필드를 추가한다.

```js
quickFacts: {
  habitat: "숲과 나무 위",
  movement: "날거나 나뭇가지를 뛰어다님",
  feature: "날개와 깃털이 있음"
}
```

기존 상세 설명은 본문으로 유지하고, `quickFacts`는 상세 설명 하단의 `<details class="observation-summary">` 안에서 접힘 패널로 제공한다.  
오답 후에는 이 패널을 별도 힌트처럼 띄우지 않고, `showHintAndScroll()`을 통해 관련 본문 문단으로 이동하고 노란색 하이라이트를 적용한다.

### 완료 기준

- 모든 동물에 `quickFacts`가 존재한다.
- 상세 모달 하단에 `관찰 요약 열기` 접힘 패널이 보인다.
- 오답 후에는 별도 힌트 패널이 아니라 본문 하이라이트가 힌트 역할을 한다.
- 퀴즈 등록 버튼은 설명 본문과 관찰 요약 뒤에 배치된다.

## 8.3 검색어 칩

### 목적

학생이 검색어를 스스로 떠올리지 못해도 탐색을 시작할 수 있게 한다.

### 권장 칩

```text
[날개] [다리] [지느러미] [물속] [땅속] [사막] [알] [털]
```

### 구현 방식

- 검색창 아래 `.search-chips` 영역 추가
- 칩 클릭 시 `searchInput.value = chipText`
- 기존 `debouncedRenderAnimals()` 호출

### 완료 기준

- 학생이 키보드를 치지 않아도 검색 결과를 필터링할 수 있다.
- 검색 결과가 없을 때 유사 칩을 제안한다.
- 모바일에서 칩이 2줄 이하로 정리된다.

## 8.4 분류게임 모바일 2단 레이아웃

### 목적

모바일에서 길게 스크롤하거나 드래그하지 않아도 분류 활동을 완료하게 한다.

### 권장 구조

```text
[카드 더미: 가로 스크롤]
🐰 🐸 🐟 🦅 🐍

[맞아요]     [달라요]
카드 영역     카드 영역
```

### 조작 방식

1. 카드를 한 번 누른다.
2. 선택된 카드에 테두리와 안내 문구가 뜬다.
3. `맞아요`, `달라요`, `카드 더미` 중 한 곳을 누른다.
4. 카드는 해당 위치로 이동한다.

### 구현 방식

- `gamePool`에 `data-answer="pool"` 추가
- `gamePool`도 drop/click handler 대상으로 포함
- 모바일에서는 drag 기능보다 tap-to-move 안내를 우선 표시
- 선택된 카드 상태를 `state.game.selectedAnimalId`로 관리

### 완료 기준

- 카드를 `yes → no`, `no → yes`, `yes/no → pool`로 모두 이동 가능하다.
- 모바일에서 드래그 없이 탭만으로 분류 가능하다.
- 채점 전 학생이 답을 수정할 수 있다.

## 8.5 외부 링크 안전 안내

### 목적

학생이 위키백과 등 외부 페이지로 이동한 뒤 수업 화면으로 돌아오지 못하는 문제를 줄인다.

### 권장 UI

```text
사진 출처 보기 🌐
새 창에서 열려요. 도감으로 돌아오려면 열린 창을 닫아 주세요.
```

또는 클릭 전 확인:

```text
사진 출처 페이지가 새 창에서 열립니다.
계속 열까요?

[계속 보기] [도감에 남기]
```

### 완료 기준

- 외부 링크에는 `target="_blank"`와 `rel="noreferrer noopener"`를 유지한다.
- 링크 옆에 “새 창” 안내가 보인다.
- 초등학생 문장 수준으로 복귀 방법을 안내한다.

## 8.6 사진 출처 일괄 보기

### 목적

공개 배포와 교사용 공유 시 사진 저작권·출처 관리 신뢰도를 높인다.

### 권장 구조

```text
사진 출처 모음

고양이
- 사진 제목:
- 저작자:
- 출처:
- 라이선스:

오리
- 사진 제목:
- 저작자:
- 출처:
- 라이선스:
```

### 구현 방식

- `credits.html` 또는 모달 내 “사진 출처 모음” 생성
- 동물 데이터에 `credit` 객체 추가
- 내장 이미지 사용 시에도 원출처 유지

### 완료 기준

- 모든 동물 이미지에 출처 정보가 연결된다.
- 학생용 화면에서는 단순하게, 교사용 화면에서는 자세히 볼 수 있다.
- 공개수업 자료나 공유 링크로 사용해도 출처 설명이 가능하다.

## 13. 최종 권고

현재 프로젝트는 이미 “자료 사이트” 수준을 넘어 **수업형 웹앱**에 가까운 완성도를 갖추고 있다.  
따라서 앞으로의 개발 방향은 다음 세 가지로 압축하는 것이 좋다.

1. **학생 화면 단순화**  
   첫 화면과 상세 화면에서 3학년 학생이 바로 행동할 수 있도록 “오늘의 지역 미션, 설명 본문, 관찰 요약, 퀴즈 등록 버튼” 중심으로 정리한다.

2. **태블릿 수업 안정화**  
   분류게임 모바일 UX, 외부 링크 복귀 안내, 공용 태블릿 기록 초기화, 이미지 내장화를 우선 처리한다.

3. **교사용 운영성 강화**  
   교사용 지역 미션 설정, 수업 리셋, 질문방 있는/없는 버전 자동 관리, 사진 출처 일괄 관리로 공개수업과 일반 수업 모두에 사용할 수 있게 한다.

가장 먼저 처리할 작업은 다음 5개다.

```text
1. 분류게임 모바일 2단 레이아웃
2. 카드 되돌리기 및 수정 기능
3. 외부 링크 새 창/복귀 안내
4. no-question 학생용 문구 정리
5. 이미지 내장 작업 착수
```

이후에는 이미 구현된 `교사용 지역 미션 설정`과 `관찰 요약` 흐름을 실제 태블릿 수업에서 검증하고, 이미지 내장·외부 링크 안내·분류게임 모바일 QA를 이어가면 초등학교 3학년 실제 수업에서 훨씬 안정적으로 사용할 수 있는 형태가 된다.
